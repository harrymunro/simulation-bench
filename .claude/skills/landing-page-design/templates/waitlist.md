# Template: Waitlist Landing Page

## Purpose
Capture interest for a product or course before launch. Build an audience to sell to on launch day.

## Best for
- Pre-launch course validation
- Beta testing signups
- "Coming soon" pages
- Early access programs

## Key Principle
**Promise + exclusivity + curiosity.** Give enough information to create desire, but not so much that waiting feels unnecessary.

## Recommended Section Order

1. Hero (what's coming + waitlist form)
2. What to expect (tease the offering)
3. Why join early (benefits of being on the list)
4. About the creator (brief credibility)
5. Secondary CTA (repeat form)

## Mobile Considerations

- Hero: Form should be visible without scrolling
- Benefits: Single column grid
- Form inputs: Full width
- Keep total page length minimal

## What NOT to Include

Waitlist pages should create desire, not satisfy it. Avoid:

- ❌ Full feature lists (save for launch)
- ❌ Pricing (even "starting at $X")
- ❌ Detailed curriculum/content outlines
- ❌ Long sales copy
- ❌ Multiple CTAs
- ❌ FAQ sections (they don't have questions yet)

## Button Copy That Works

- "Join the waitlist"
- "Notify me"
- "Get early access"
- "Save my spot"
- "Count me in"

## Post-Signup Experience

Consider what happens after signup:

```
┌─────────────────────────────────────────────────────────────────┐
│                   [CONFIRMATION PAGE]                           │
│                                                                 │
│                       You're in! ✓                              │
│                                                                 │
│              "You're #[X] on the waitlist"                      │
│                                                                 │
│                    What happens next:                           │
│                                                                 │
│   1. You'll get an email confirming your spot                   │
│   2. We'll send occasional updates on progress                  │
│   3. You'll be first to know when we launch                     │
│                                                                 │
│        ┌─────────────────────────────────────┐                  │
│        │  [SHARE WITH OTHERS - OPTIONAL]     │                  │
│        └─────────────────────────────────────┘                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

> **Design note**: The confirmation page is a missed opportunity
> on most waitlist pages. Use it to build relationship.
